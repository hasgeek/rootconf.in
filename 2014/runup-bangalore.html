Tools & Methods for Cloud Automation

5 April 2014   Inmobi, Bellandur, Bangalore

Using automation to manage large-scale infrastructure is the key to making an organisation and product more efficient and cost-effective. At this event, our featured speakers explain how they achieved automation using tools like Puppet, Hiera, Mesos, Docker and AWS.

Cloud infrastructure management has become an integral part of a DevOps or Systems Engineer's role. Though the rise of virtualization has brought a lot of flexibility in managing costs, it has also added complexity by increasing the number of servers to manage. Provisioning and deployment automation is the only way this can be managed now. This meet is intended to address these concerns in the daily life of a cloud/web operations team. It covers 3 talks around this using tools like Docker, Chef, Jenkins, Puppet and Vagrant!

Talks

Roles and Profiles in Puppet: Yet Another Example

Jasbir solved the problem of constant infrastructure change requests he received from developers, by abstracting them as configurations into YAML files that developers can modify themselves. The talk revolves around using Hiera and examples of roles and profiles in puppet.

Jasbir Khehra

Head Infrastructure, NDTV Convergence

Mesos + Docker = ♡

The objective of the talk is to bounce off some ideas on what is possible with Mesos and Docker. Abishek will introduce the core ideas behind Mesos and Docker, and how they can be used together to build a completely elastic cluster. The structure of the talk can be found here.

Abishek works with Airwoot as a Machine Learning/Platform Engineer and is interested in Large Scale Data Processing Infrastructures and Mathematics!

Abishek Bhat

Machine Learning/Platform Engineer, Airwoot

Automate infrastructure deployment with Amazon Web Services

As startups focus more and more towards innovating new stuff and creating value for their customer less they want to be bothered about infrastructure and related heavy lifting. This is where AWS helps startups to leverage Infrastructure automation and deployment services like Elastic Beanstalk, Cloudformation and OpsWorks. Learn the difference between these three powerful services and how it helps startups.

Amit likes helping AWS customers of all sizes ranging from startups to big enterprises to hop onto the Cloud phenomenon and realize the benefits. Interestingly, he has been spending his free time these days on Arduinos and RaspberryPis.

Amit Sharma

Solutions Architect, Amazon Seller Services Pvt. Ltd.
